RAG (Retrieval-Augmented Generation)

8 focus blocks~4 hours totalPhase 5 prep — DAI-01

What is RAG?"Retrieval-Augmented Generation" — instead of asking an LLM from its memory (which hallucinates), you give it relevant chunks of documents to read, then ask. Used in every "chat with your docs" product (Notion AI, GitHub Copilot Chat, etc.). Mini-project: a RAG API that answers questions about your own notes.

Use ChromaDB (embedded vector database) to keep things simple. No external services beyond the LLM.

The full pipeline

flowchart LR subgraph index[Indexing - one time] A[Documents] --> B[Chunk] --> C[Embed] --> D[(Vector DB)] end subgraph query[Query - per request] E[Question] --> F[Embed] --> G[Search top-k] --> H[Retrieved chunks] --> I[LLM with chunks] --> J[Answer + citations] end D -.-> G

Embeddings concept

Create 01-embeddings.py

import os, numpy as np
import google.generativeai as genai
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

sentences = [
    "I love my dog",
    "My puppy is the best companion",
    "The stock market crashed yesterday",
    "Federal Reserve raised interest rates",
    "Python is a great programming language",
]

def embed(text):
    return genai.embed_content(model="models/text-embedding-004", content=text)["embedding"]

vectors = [embed(s) for s in sentences]

def cosine(a, b):
    a, b = np.array(a), np.array(b)
    return (a @ b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(f"dog vs puppy:        {cosine(vectors[0], vectors[1]):.3f}")
print(f"dog vs stocks:       {cosine(vectors[0], vectors[2]):.3f}")
print(f"stocks vs Fed:       {cosine(vectors[2], vectors[3]):.3f}")
Worked? Semantic pairs (dog/puppy, stocks/Fed) score higher than unrelated.

Chunking strategies

Create 02-chunking.py

import requests

# Grab a Wikipedia article as test data
text = requests.get("https://en.wikipedia.org/wiki/Sheffield?action=raw").text[:50000]

def fixed_chunker(text, size=500, overlap=50):
    chunks = []
    for i in range(0, len(text), size - overlap):
        chunks.append(text[i:i+size])
    return chunks

def paragraph_chunker(text):
    return [p for p in text.split("\n\n") if p.strip()]

print("fixed:    ", len(fixed_chunker(text)), "chunks")
print("paragraph:", len(paragraph_chunker(text)), "chunks")
Worked? Two different chunk counts. You understand chunks let LLMs work within their context limit.

Vector DB index

Create 03-index.py

import chromadb
import requests
from app_embed import embed   # we'll create this

client = chromadb.PersistentClient(path="./chroma_db")
collection = client.get_or_create_collection("notes")

text = requests.get("https://en.wikipedia.org/wiki/Sheffield?action=raw").text[:50000]
chunks = [text[i:i+500] for i in range(0, len(text), 450)][:50]

collection.add(
    documents=chunks,
    ids=[f"chunk-{i}" for i in range(len(chunks))],
    embeddings=[embed(c) for c in chunks],
    metadatas=[{"source": "wikipedia/Sheffield", "chunk_index": i} for i in range(len(chunks))],
)

print(f"indexed {collection.count()} chunks")

Retrieval

Create 04-retrieve.py

import chromadb
from app_embed import embed

client = chromadb.PersistentClient(path="./chroma_db")
collection = client.get_collection("notes")

def retrieve(question, k=3):
    q_embedding = embed(question)
    return collection.query(query_embeddings=[q_embedding], n_results=k)

for q in ["What is Sheffield famous for?", "Tell me about the population", "What's the weather on Mars?"]:
    print(f"\nQ: {q}")
    results = retrieve(q)
    for doc, dist in zip(results["documents"][0], results["distances"][0]):
        print(f"  ({dist:.3f}) {doc[:80]}...")
Worked? First two questions return relevant chunks; the Mars question gets distant matches.

Generation with grounding

Create 05-generate.py

import google.generativeai as genai
import os
from dotenv import load_dotenv
import chromadb
from app_embed import embed

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))
client = chromadb.PersistentClient(path="./chroma_db")
collection = client.get_collection("notes")
model = genai.GenerativeModel("gemini-2.5-flash-lite")

def ask(question, k=3):
    res = collection.query(query_embeddings=[embed(question)], n_results=k)
    chunks = res["documents"][0]
    context = "\n\n---\n\n".join(f"[chunk {i}]\n{c}" for i, c in enumerate(chunks))
    prompt = f"""Answer using ONLY the context below.
If the answer isn't in the context, say "I don't know".
Cite the chunk numbers you used.

Context:
{context}

Question: {question}"""
    return model.generate_content(prompt).text

print(ask("What is Sheffield famous for?"))
print("---")
print(ask("Has anyone landed on Mars?"))
Worked? First answer is grounded with citations; second says "I don't know" rather than hallucinate.

Evals

Create 06-eval.py

from app_generate import ask   # extract ask() into app_generate.py first

eval_set = [
    ("What region is Sheffield in?", "South Yorkshire"),
    ("What is Sheffield's population?", "population"),
    ("What is the capital of France?", "I don't know"),
]

results = []
for q, expected in eval_set:
    answer = ask(q).lower()
    passed = expected.lower() in answer
    results.append((q, passed, answer[:80]))
    print(f"{'PASS' if passed else 'FAIL'}: {q}")
    print(f"  -> {answer[:120]}\n")

accuracy = sum(1 for _, p, _ in results if p) / len(results)
print(f"\naccuracy: {accuracy:.1%}")

Wrap as a FastAPI endpoint

Create app/main.py

from fastapi import FastAPI
from pydantic import BaseModel
from app_generate import ask
import chromadb
from app_embed import embed

app = FastAPI()
client = chromadb.PersistentClient(path="./chroma_db")
collection = client.get_collection("notes")

class Question(BaseModel):
    text: str

@app.post("/ask")
def answer(q: Question):
    res = collection.query(query_embeddings=[embed(q.text)], n_results=3)
    citations = [
        {"chunk_index": m["chunk_index"], "source": m["source"], "score": float(d)}
        for m, d in zip(res["metadatas"][0], res["distances"][0])
    ]
    return {"answer": ask(q.text), "citations": citations}
