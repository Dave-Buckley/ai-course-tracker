Type hints

What's a type hint?Python doesn't enforce types at runtime, but you can annotate them in code so tools (mypy, your IDE) catch mistakes before you run. Hiring managers expect typed Python in production.

1. 08-basic-hints.py — annotate three functions

Create. Type:

def double(n: int) -> int:
    return n * 2

def shout(msg: str) -> str:
    return msg.upper() + "!"

def squares(upto: int) -> list[int]:
    return [i * i for i in range(upto)]

print(double(5))
print(shout("hello"))
print(squares(5))

Run.

Debugging

1. 10-print-debug.py — find a bug with prints

Create. Type (the bug is on purpose):

def total(nums):
    s = 0
    for i in range(len(nums) + 1):  # bug: off-by-one
        s += nums[i]
    return s

print(total([1, 2, 3]))

Run. Crash. Now add a print inside the loop:

def total(nums):
    s = 0
    for i in range(len(nums) + 1):
        print(f"i={i}, len={len(nums)}")
        s += nums[i]
    return s

print(total([1, 2, 3]))

Run again — you see i=3, len=3 just before the crash. Fix it: change + 1 to nothing.

Worked? After the fix, prints 6 cleanly.

3. 11-logging.py — replace prints with proper logging

Why: in real apps, sprinkled prints become noise. Logging lets you set levels (debug, info, warning, error) and filter.

Create. Type:

import logging
logging.basicConfig(level=logging.DEBUG, format="%(levelname)s: %(message)s")

def process(items):
    logging.info(f"processing {len(items)} items")
    for item in items:
        logging.debug(f"item: {item}")
    logging.warning("almost done")
    return len(items)

print(process(["a", "b", "c"]))

Run.

Worked? Output shows DEBUG, INFO, WARNING tags before each message.

2. 18-timer-decorator.py — your own decorator

Why: a decorator wraps a function with extra behaviour. Used in fastapi, pytest, fast.ai constantly.

Create. Type:

import time
from functools import wraps

def timer(fn):
    @wraps(fn)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = fn(*args, **kwargs)
        elapsed = time.time() - start
        print(f"{fn.__name__} took {elapsed:.3f}s")
        return result
    return wrapper

@timer
def slow_add(a, b):
    time.sleep(1)
    return a + b

print(slow_add(2, 3))

Run.

Worked? Prints slow_add took 1.001s then 5.

Generators

What's a generator?A function that yields values one at a time instead of returning a full list. Memory-efficient for huge sequences. fast.ai DataLoaders are generators.

1. 19-yield.py — count-up generator

Create. Type:

def count_up(n):
    for i in range(n):
        yield i * i

for value in count_up(5):
    print(value)

Run.

Worked? Prints 0, 1, 4, 9, 16. Each one was generated lazily.

Mini-project setup: task-tracker.py

1. Create task-tracker.py

In VS Code: New File → task-tracker.py. Type:

from datetime import datetime
import json
from pathlib import Path

TASKS_FILE = Path("tasks.json")

class Task:
    def __init__(self, id: int, title: str, done: bool = False) -> None:
        self.id = id
        self.title = title
        self.created_at = datetime.now().isoformat()
        self.done = done

    def __str__(self) -> str:
        check = "[x]" if self.done else "[ ]"
        return f"{check} {self.id}: {self.title}"

def load_tasks() -> list[Task]:
    if not TASKS_FILE.exists():
        return []
    raw = json.loads(TASKS_FILE.read_text())
    return [Task(t["id"], t["title"], t["done"]) for t in raw]

def save_tasks(tasks: list[Task]) -> None:
    raw = [{"id": t.id, "title": t.title, "done": t.done} for t in tasks]
    TASKS_FILE.write_text(json.dumps(raw, indent=2))

Save.

CLI commands + ship

1. Replace __main__ with a CLI loop

Replace the if __name__ == "__main__": block with:

if __name__ == "__main__":
    tasks = load_tasks()
    while True:
        cmd = input("> ").strip().split(maxsplit=1)
        if not cmd: continue
        if cmd[0] == "list":
            for t in tasks: print(t)
        elif cmd[0] == "add" and len(cmd) == 2:
            new_id = max((t.id for t in tasks), default=0) + 1
            tasks.append(Task(new_id, cmd[1]))
            save_tasks(tasks)
            print(f"added: {tasks[-1]}")
        elif cmd[0] == "done" and len(cmd) == 2:
            tid = int(cmd[1])
            for t in tasks:
                if t.id == tid: t.done = True
            save_tasks(tasks)
        elif cmd[0] == "delete" and len(cmd) == 2:
            tid = int(cmd[1])
            tasks = [t for t in tasks if t.id != tid]
            save_tasks(tasks)
        elif cmd[0] == "quit":
            break
        else:
            print("commands: list / add <text> / done <id> / delete <id> / quit")

Save. Run.
