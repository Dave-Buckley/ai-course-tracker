TypeScript

8 focus blocks~4 hours totalPhase 3 prepReference: TS Handbook

What is TypeScript? TypeScript is JavaScript plus a type system. Same runtime, same code (mostly), but you annotate variables with types and a compiler catches mistakes before you run. The Vercel AI SDK and Next.js are written in TypeScript — Phase 3 needs this.

Mini-project at the end is a small typed utility library you push to GitHub.

Generics

What's a generic?A type variable that lets a function or interface work with many types while preserving type safety. Like a placeholder you fill in at the call site.

1. 11-generic-fn.ts

function identity<T>(x: T): T {
  return x;
}

console.log(identity<string>("hello"));
console.log(identity<number>(42));
// TS often infers T, so identity("auto") works too

2. 12-generic-array.ts

function first<T>(arr: T[]): T | undefined {
  return arr.length > 0 ? arr[0] : undefined;
}
console.log(first([1, 2, 3]));
console.log(first<string>([]));

3. 13-constrained-generic.ts

function logLength<T extends { length: number }>(x: T): T {
  console.log("length:", x.length);
  return x;
}
logLength("hello");
logLength([1, 2, 3]);
// logLength(42); // error - number has no length

Narrowing + discriminated unions

1. 14-typeof-narrow.ts

function format(value: string | number): string {
  if (typeof value === "string") {
    return value.toUpperCase();   // TS knows it's a string here
  }
  return value.toFixed(2);        // TS knows it's a number here
}
console.log(format("hello"));
console.log(format(3.14159));

2. 15-discriminated-union.ts — the cleanest pattern

type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "square"; side: number };

function area(s: Shape): number {
  switch (s.kind) {
    case "circle": return Math.PI * s.radius ** 2;
    case "square": return s.side ** 2;
  }
}
console.log(area({ kind: "circle", radius: 5 }));
console.log(area({ kind: "square", side: 4 }));

Utility types

1. 16-utility.ts — Partial, Pick, Omit, Record

type User = {
  id: number;
  name: string;
  email: string;
  password: string;
};

type UserPatch = Partial<User>;       // all fields optional
type PublicUser = Omit<User, "password">;
type UserSummary = Pick<User, "id" | "name">;
type UserMap = Record<string, User>;

const patch: UserPatch = { name: "Alice" };
const summary: UserSummary = { id: 1, name: "Bob" };
console.log(patch, summary);
Worked? Runs cleanly. You can derive shapes from a base type without writing them out.

Tiny typed utility library

1. Create lib.ts with three exports

Type:

export function groupBy<T, K extends string | number>(
  arr: T[],
  key: (item: T) => K
): Record<K, T[]> {
  const out = {} as Record<K, T[]>;
  for (const item of arr) {
    const k = key(item);
    (out[k] ||= []).push(item);
  }
  return out;
}

export function pick<T, K extends keyof T>(obj: T, keys: K[]): Pick<T, K> {
  const out = {} as Pick<T, K>;
  for (const k of keys) out[k] = obj[k];
  return out;
}

export function safeJsonParse<T>(s: string): T | null {
  try { return JSON.parse(s) as T; } catch { return null; }
}

2. lib.test.ts — tests

import { groupBy, pick, safeJsonParse } from "./lib";

const people = [
  { name: "Alice", city: "London" },
  { name: "Bob", city: "Sheffield" },
  { name: "Carol", city: "London" },
];

const grouped = groupBy(people, (p) => p.city);
console.assert(grouped.London.length === 2, "groupBy failed");

const picked = pick({ a: 1, b: 2, c: 3 }, ["a", "c"]);
console.assert(picked.a === 1 && (picked as any).b === undefined, "pick failed");

console.assert(safeJsonParse<number>("42") === 42, "json parse failed");
console.assert(safeJsonParse("not json") === null, "json fail handling failed");

console.log("all tests passed");

npx ts-node lib.test.ts
