Routing — pages, dynamic routes, links

App Router ruleFolders inside src/app/ become URL paths. A page.tsx file inside renders the page. Square brackets in folder names = dynamic params.

1. New route: /about

In VS Code, create folder src/app/about/, then file src/app/about/page.tsx. Type:

export default function AboutPage() {
  return <h1>About me</h1>;
}

Visit localhost:3000/about.

2. Blog index /blog

Create src/app/blog/page.tsx. Type:

import Link from "next/link";

export default function BlogIndex() {
  return (
    <div>
      <h1>Blog</h1>
      <ul>
        <li><Link href="/blog/post-1">Post 1</Link></li>
        <li><Link href="/blog/post-2">Post 2</Link></li>
      </ul>
    </div>
  );
}

3. Dynamic route /blog/[slug]

Create folder src/app/blog/[slug]/, then file src/app/blog/[slug]/page.tsx. Type:

export default async function PostPage({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;
  return <h1>Post: {slug}</h1>;
}

Visit /blog/post-1 — see the slug rendered.

Worked? All four routes work, links don't cause full reloads.

Server vs Client components

The ruleEverything is a Server Component by default — runs on the server, can fetch data, can read files, but can't use useState or click handlers. Add "use client" at the top to opt into the client (with state, effects, browser APIs).

1. A Client Component: Counter.tsx

Create folder src/components/, then file src/components/Counter.tsx. Type:

"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(count + 1)} className="border px-4 py-2 rounded">
      Count: {count}
    </button>
  );
}

2. Use it on the homepage

Edit src/app/page.tsx:

import Counter from "@/components/Counter";

export default function Home() {
  return (
    <main className="p-4">
      <h1 className="text-3xl font-bold">Home</h1>
      <Counter />
    </main>
  );
}
Worked? Click the button — count increments in the browser. Try removing "use client" from Counter.tsx — observe the runtime error explaining why.

Data fetching in Server Components

1. /posts — fetch a list

Create src/app/posts/page.tsx. Type:

type Post = { id: number; title: string };

export default async function Posts() {
  const res = await fetch("https://jsonplaceholder.typicode.com/posts", {
    next: { revalidate: 60 }
  });
  const posts: Post[] = await res.json();
  return (
    <ul className="p-4 space-y-2">
      {posts.slice(0, 10).map(p => (
        <li key={p.id}>
          <a href={`/posts/${p.id}`} className="underline">{p.title}</a>
        </li>
      ))}
    </ul>
  );
}

revalidate: 60 means Next caches the response for 60 seconds before re-fetching.

2. /posts/[id] — single post

Create src/app/posts/[id]/page.tsx. Type:

export default async function Post({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params;
  const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
  const post = await res.json();
  return (
    <article className="p-4">
      <h1 className="text-2xl font-bold">{post.title}</h1>
      <p className="mt-2">{post.body}</p>
    </article>
  );
}
Worked? /posts lists titles, clicking opens individual posts.

Server Actions

What's a Server Action?A function marked "use server" that runs on the server but can be called from a client form. Replaces traditional API routes for most mutations.

1. Install Zod for validation

npm install zod

2. Create src/app/actions.ts

"use server";
import { z } from "zod";
import { revalidatePath } from "next/cache";

const Schema = z.object({ title: z.string().min(3) });

export async function createPost(formData: FormData) {
  const parsed = Schema.parse({ title: formData.get("title") });
  console.log("would save:", parsed);
  revalidatePath("/posts");
}

3. Use it from a form

Edit src/app/posts/page.tsx — add at the top:

import { createPost } from "@/app/actions";

And in the JSX, add a form:

<form action={createPost} className="p-4 flex gap-2">
  <input name="title" className="border px-2 py-1" placeholder="Post title..." />
  <button type="submit" className="border px-4 py-1 rounded">Add</button>
</form>
Worked? Submit the form. The console (where npm run dev is running) shows would save: { title: "..." }.

Metadata + SEO

1. Static metadata

In any page.tsx, export:

import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "Posts",
  description: "All posts in the blog",
};

2. Dynamic metadata for the post page

export async function generateMetadata({ params }: { params: Promise<{ id: string }> }): Promise<Metadata> {
  const { id } = await params;
  const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
  const post = await res.json();
  return { title: post.title };
}
Worked? View page source — each route has a unique <title> tag.
